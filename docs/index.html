▸ 什么是 Agentic AI？

Agentic AI（智能体 AI）是当前 AI 领域最重要的范式转变——从「对话」到「行动」。

▸ 什么是智能体？

传统 AI 是「问答式」的——你问它答，输出停留在文字。而智能体（Agent）是「行动式」的——它能调用工具、操作文件、访问 API、执行代码，真正改变你电脑里的状态。

# 传统 AI：只回答
你：帮我查一下今天的天气
→ "抱歉，我无法获取实时信息"

# 智能体 AI：直接做事
你：帮我查一下今天的天气
→ 调用天气 API → 获取数据 → 返回结果
✓ 北京 25°C，晴，适合出门

▸ 核心三要素

一个完整的 AI 智能体由三个核心组件构成：

1 大语言模型（LLM）

大脑——理解自然语言、推理决策、规划步骤。支持 DeepSeek、OpenAI、Ollama 等 7 种提供商。

2 工具系统（Tools）

手脚——读写文件、搜索代码、调用 API、操作数据库。每个工具都是智能体与真实世界交互的接口。

3 记忆系统（Memory）

记忆——长期记忆保存偏好和规则，短期记忆跟踪当前任务上下文，让对话有连续性。

▸ 智能体 vs 聊天机器人

维度聊天机器人智能体
输出文字文字 + 行动
工具无文件、API、代码、搜索
记忆单轮对话长期 + 短期记忆
自主性被动回答主动规划执行
影响范围聊天窗口内你的数字世界

▸ 为什么现在？

2024-2025 年，三个趋势交汇催生了智能体的爆发：

  • ▸ LLM 能力跃升 — 推理、规划、工具调用能力大幅提升
  • ▸ 工具标准化 — MCP、Function Calling 等协议让 AI 能统一调用工具
  • ▸ 开源生态成熟 — 本地模型、低成本 API 让每个人都能运行智能体
Bobo 的定位：一个开源的、运行在你终端里的通用智能体。不是 SaaS，不是云服务——是你的个人 AI 助手，完全由你掌控。

▸ 工具系统：智能体的手脚

LLM 是大脑，工具就是手脚。没有工具的 AI 只能思考，有了工具的 AI 才能行动。

▸ 什么是工具系统？

工具系统是智能体与真实世界交互的接口。每个工具都是一个函数——有输入参数、有输出结果。智能体根据任务需求，自主选择调用哪个工具、传入什么参数。

# Bobo 的工具调用示例
你：帮我搜索笔记中关于"智能体"的内容
→ Bobo 调用 search_obsidian(query="智能体")
✓ 找到 3 篇相关笔记

你：把结果保存到新笔记
→ Bobo 调用 write_obsidian(filename, content)
✓ 笔记已创建

▸ 工具的三种类型

智能体的工具可以分为三个层次：

1 内置工具

读写文件、搜索代码、执行终端命令、操作 Obsidian 笔记——Bobo 出厂自带的核心能力。

2 API 工具

通过 api_register 连接任何 REST API——Jira、飞书、GitHub、自定义后端。无需写代码，一条指令注册一个 API。

3 MCP 协议

Model Context Protocol——AI 界的 USB 标准。任何实现了 MCP 的服务都可以无缝接入 Bobo 的工具系统。

▸ 工具调用的生命周期

一个完整的工具调用经历五个阶段：

# 1. 意图识别
Bobo: 用户想搜索笔记 → 需要 search_obsidian 工具

# 2. 参数构建
Bobo: search_obsidian(query="智能体入门")

# 3. 工具执行
系统: 执行中...

# 4. 结果解析
Bobo: 返回了 3 条结果，格式正确

# 5. 反馈用户
✓ 已找到 3 篇相关笔记，以下是摘要...

▸ 为什么工具系统是核心？

没有工具的 AI 只是一个知识库，有了工具的 AI 才是一个智能体。

  • ▸ 突破信息边界 — 不再局限于训练数据，能实时获取最新信息
  • ▸ 改变真实世界 — 创建文件、发送邮件、管理项目，产生实际影响

Bobo 目前拥有 30+ 个内置工具，覆盖文件操作、代码开发、知识管理、通信协作、系统控制等类别。
▸ 记忆系统：让 AI 记住你

没有记忆的 AI 每次对话都是陌生人。记忆系统让智能体认识你、了解你、记住你的偏好。

▸ 为什么需要记忆？

大语言模型本身是无状态的——每次对话都是独立的，它不记得上次你说了什么。记忆系统就是为了解决这个问题而设计的。

# 没有记忆的 AI
你：我叫小明
→ "你好小明！"
你：我叫什么名字？
→ "抱歉，我不知道你的名字"

# 有记忆的 AI（Bobo）
你：我叫小明
→ "你好小明！"
你：我叫什么名字？
→ "你叫小明"
✓ 记忆已保存，下次对话仍有效

▸ 记忆的层次

Bobo 的记忆系统分为三个层次：

1 短期记忆

当前对话的上下文——你刚才说了什么、正在做什么任务。对话结束后自动清空。

2 长期记忆

通过 save_memory 保存的永久信息——你的名字、偏好、工作习惯。每次对话自动注入。

3 知识库

你的 Obsidian 笔记库——Bobo 可以搜索、阅读、归类你的笔记，把你的知识变成它的知识。

▸ 记忆的实际应用

记忆系统让 Bobo 能做到很多普通 AI 做不到的事：

  • ▸ 记住你的名字和偏好 — "我喜欢喝冰美式" → 以后每次提到咖啡都知道
  • ▸ 记住工作规则 — "修改代码必须用 diff 格式" → 每次改代码都自动遵守
  • ▸ 记住项目上下文 — "继续昨天的工作" → 自动定位到上次的文件
  • ▸ 跨会话连续性 — 关闭终端再打开，Bobo 还记得你

▸ 隐私设计

记忆系统存储在你的本地机器上，不会上传到任何云端。你可以随时查看、编辑或删除记忆。

Bobo 的记忆原则：你告诉它的事它记住，你没说的它不问。记忆完全本地存储，你可以随时清空。
▸ 从对话到行动：智能体的工作流

真正的智能体不是被动回答，而是主动规划、执行、验证——像一个工程师一样工作。

▸ 智能体的思考循环

智能体不是一次调用就结束的。它遵循一个思考-行动-观察的循环：

# 思考循环示例：修复一个 bug
你：帮我修复这个 bug

# 第1轮：思考 → 行动 → 观察
Bobo: 先搜索相关代码
→ grep_code("bug_pattern")
✓ 找到 3 处匹配

# 第2轮：思考 → 行动 → 观察
Bobo: 读取文件确认问题
→ read_local_file("server.py")
✓ 发现第42行有逻辑错误

# 第3轮：思考 → 行动 → 观察
Bobo: 修复并验证
→ edit_file → run_tests
✓ 测试通过，bug 已修复

▸ 智能体的核心能力

一个成熟的智能体具备以下能力：

1 任务分解

把一个大目标拆成多个小步骤，按顺序执行。例如"帮我搭建一个博客"→ 创建项目 → 安装依赖 → 编写代码 → 启动服务。

2 错误恢复

工具调用失败时自动尝试替代方案。API 超时 → 换一个端点；文件不存在 → 搜索相似文件。

3 自我验证

修改代码后自动运行测试，确认没有破坏已有功能。写文件后读取验证内容正确。

▸ 常用工作流 · 专栏

几个典型场景，展示 Bobo 如何帮你提效。

▸ 修改代码 → 测试 → PR

1

定位代码

用 grep_code 搜索函数定义或关键词，精确找到要修改的位置。

2

精确修改

用 edit_file 做字符串替换，只改目标代码，不动整体架构。变更用 diff 格式展示。

3

运行测试

自动运行测试套件，失败则定位问题 → 修复 → 再次验证，直到全部通过。

4

创建 PR

从当前分支创建 Pull Request，附带变更说明，一键提交审查。

$ grep_code("def calculate")
→ src/pricing.py:42

$ edit_file(
  old="tax_rate = 0.13",
  new="tax_rate = 0.15"
)
✓ 已替换

$ run_tests()
✓ 12 passed, 0 failed

$ github_create_pr(
  title="fix: update tax rate"
)
✓ PR #42 created
